# GitHub Actions Firebase Hosting Deployment Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create GitHub Actions workflows for mobile_app to automatically build and deploy Flutter Web to Firebase Hosting on push or PR.

**Architecture:** Add two GitHub Action workflows targeting the Firebase project `trippo-73fd7`. One will deploy to the live channel on push/merge to `main`, and the other will deploy to the preview channel on pull requests.

**Tech Stack:** GitHub Actions, Firebase Hosting, Flutter Web.

## Global Constraints
- Target Firebase Project ID is `trippo-73fd7`.
- Target GitHub Secrets must contain `FIREBASE_SERVICE_ACCOUNT_TRIPPO_73FD7`.
- Build web release output folder is `build/web`.

### Task 1: Create Merge Workflow File
Create the workflow file that triggers on merge or push to `main` and deploys to the live Firebase Hosting channel.

**Files:**
- Create: `.github/workflows/firebase-hosting-merge.yml`

**Interfaces:**
- Produces: GitHub Action workflow executing `flutter build web` and `firebase deploy` on main.

- [ ] **Step 1: Write the merge workflow file**

Write the following content to `.github/workflows/firebase-hosting-merge.yml`:
```yaml
# This file was auto-generated by the Firebase CLI
# https://github.com/firebase/firebase-tools

name: Deploy to Firebase Hosting on merge
on:
  push:
    branches:
      - main
jobs:
  build_and_deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: subosito/flutter-action@v2
        with:
          channel: 'stable'
      - run: cp env.json.example env.json
      - run: flutter build web
      - uses: FirebaseExtended/action-hosting-deploy@v0
        with:
          repoToken: ${{ secrets.GITHUB_TOKEN }}
          firebaseServiceAccount: ${{ secrets.FIREBASE_SERVICE_ACCOUNT_TRIPPO_73FD7 }}
          channelId: live
          projectId: trippo-73fd7
```

- [ ] **Step 2: Commit changes**
Run:
```bash
git add .github/workflows/firebase-hosting-merge.yml
git commit -m "feat: add github action workflow for firebase hosting merge deploy"
```

### Task 2: Create Pull Request Workflow File
Create the workflow file that triggers on pull requests to `main` and deploys to a preview channel.

**Files:**
- Create: `.github/workflows/firebase-hosting-pull-request.yml`

**Interfaces:**
- Produces: GitHub Action workflow executing `flutter build web` and `firebase preview` on pull request.

- [ ] **Step 1: Write the pull request workflow file**

Write the following content to `.github/workflows/firebase-hosting-pull-request.yml`:
```yaml
# This file was auto-generated by the Firebase CLI
# https://github.com/firebase/firebase-tools

name: Deploy to Firebase Hosting on PR
on: pull_request
permissions:
  checks: write
  contents: read
  pull-requests: write
jobs:
  build_and_preview:
    if: ${{ github.event.pull_request.head.repo.full_name == github.repository }}
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: subosito/flutter-action@v2
        with:
          channel: 'stable'
      - run: cp env.json.example env.json
      - run: flutter build web
      - uses: FirebaseExtended/action-hosting-deploy@v0
        with:
          repoToken: ${{ secrets.GITHUB_TOKEN }}
          firebaseServiceAccount: ${{ secrets.FIREBASE_SERVICE_ACCOUNT_TRIPPO_73FD7 }}
          projectId: trippo-73fd7
```

- [ ] **Step 2: Commit changes**
Run:
```bash
git add .github/workflows/firebase-hosting-pull-request.yml
git commit -m "feat: add github action workflow for firebase hosting pull request preview"
```
